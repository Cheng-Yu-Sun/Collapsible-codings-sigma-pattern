# Collapsible Codings and the Sigma Pattern

This repository contains the R code and data files for reproducing the numerical results and regular-design searches in the paper:

**A Unified Framework for Multi-Scale Stratification in Space-Filling Designs via Collapsible Codings**

## Required R packages

The following R packages are required:

```r
install.packages(c(
  "doSNOW", "foreach", "SLHD", "MaxPro",
  "LHD", "TestFunctions", "hetGP"
))
```

The package `parallel` is also used, but it is included with base R.

## Working directory

Before running the scripts, please set the working directory to the repository root.

The repository root is the folder containing `README.md`, `R/`, and `data/`.

Some scripts include a simple check that moves the working directory back to the repository root if they are run from a subfolder.

## Constructing the fixed designs

The two fixed four-level arrays used in the simulation study are stored in:

```text
data/D_Sigma.rds
data/D_SP.rds
```

They can be regenerated by running:

```r
source("R/construct_designs/construct_D_Sigma.R")
source("R/construct_designs/construct_D_SP.R")
```

Repeated executions may return different arrays if the random seed is changed or removed, but the resulting designs have the same Sigma/SP patterns.

## Reproducing Figure 2

Figure 2 corresponds to the self-defined test function with `lambda = 30` and `lambda = 3`.

Run:

```r
source("R/run_self_defined_test_function.R")
```

This script saves the NRMSE results in `results/` and the generated boxplots in `outputs/figures/`.

## Reproducing Figure 3

Figure 3 corresponds to the OTL Circuit, Borehole, Steel Column, and Wing Weight test functions.

Run:

```r
source("R/run_additional_test_functions.R")
```

This script saves the NRMSE results in `results/` and the generated boxplots in `outputs/figures/`.

## Reproducing Table 1

Table 1 reports Sigma-optimal 4-level regular designs with 16 runs for m = 3, 4, 5.

Run:

```r
source("R/Table1_regular_4level_16run.R")
```

The final results are saved in `outputs/tables/`.

## Reproducing Table 2

Table 2 reports Sigma-optimal 4-level regular designs with 64 runs for m = 7, ..., 16.

Run:

```r
source("R/Table2_regular_4level_64run.R")
```

The final results are saved in `outputs/tables/`.

Intermediate files are saved in `outputs/Table2_intermediate/`.

## Reproducing Table 3

Table 3 reports Sigma-optimal or nearly Sigma-optimal 9-level regular designs with 81 runs for m = 3, ..., 10.

Run:

```r
source("R/Table3_regular_9level_81run.R")
```

For m = 3, 4, 5, the designs are obtained by exhaustive search with complete Sigma-pattern comparison.

For m = 6, ..., 10, the designs are obtained by the partial iterative procedure described in the Supplement and are reported as nearly Sigma-optimal.

The final results are saved in `outputs/tables/`.

Intermediate files are saved in `outputs/Table3_intermediate/`.

## Notes on computation

The simulation scripts may take some time because Gaussian process models are fitted repeatedly.

The regular-design search scripts, especially those for Tables 2 and 3, may also be computationally intensive. Intermediate `.rds` files are saved to avoid losing search results during long runs.
